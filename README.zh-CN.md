# ElBot

中文 | [English](README.md)

ElBot 是一个使用 Go 编写的轻量级 Agent/Chatbot 框架，目标是在保留可扩展性的同时，尽量降低运行成本、上下文成本和维护复杂度。它支持普通聊天、工具调用、Hook 扩展、长期任务调度、持久化会话与上下文压缩，适合个人助理、平台机器人和可编排自动化助手等场景。

## 特色

### 0. 轻量高效的 Go 实现

ElBot 使用 Go 语言实现，启动快、部署简单、资源占用低。当前本地启动耗时约 2ms，常驻内存约 20MB，适合长期运行在个人电脑、低配服务器或轻量容器中。相比依赖复杂运行时的大型 Agent 框架，ElBot 更强调可控、稳定和易维护。

### 1. 极致节省 Token 的工具发现机制

研究表明，许多普通用户仍主要将 ChatGPT 类产品用作更高级的搜索引擎、写作助手和倾听对象，频繁工具调用并不是所有对话的常态。ElBot 不会在每轮对话中默认注入所有工具的完整 schema，而是仅暴露 `discover_tool` 和当前可用工具名称。模型需要使用工具时，先按需发现工具详情，再由 Agent 注入对应 schema。这种设计减少了绝大多数普通聊天中的无效上下文开销。

参考：Chatterji et al., *How People Use ChatGPT*, NBER, 2025；Yan et al., *ShareChat: A Dataset of Chatbot Conversations in the Wild*, arXiv:2512.17843, 2025。

### 2. Chat / Work 双模式

ElBot 区分 chat 模式与 work 模式。chat 模式完全移除工具，更适合日常闲聊、陪伴、轻量问答和低成本对话；work 模式则启用工具发现与工具调用能力，用于搜索、执行命令、创建任务、读写 skill 等复杂工作。两种模式可以独立配置模型，让低成本模型承担闲聊，让强模型专注处理复杂任务。

### 3. 可扩展的 Hook 系统

ElBot 内置 Hook Layer，可在 Agent 输入、LLM 请求、LLM 响应、平台发送、平台连接等关键事件点插入扩展逻辑。Hook 可以修改消息、追加输出意图、调用低风险工具或注入常驻记忆。内置规则 Hook、表情 Hook 和常驻记忆 Hook，也支持后续扩展独立插件，让核心逻辑保持稳定，个性化能力放在插件层演进。

### 4. 完善的日志与审计系统

ElBot 区分普通运行日志与审计日志，支持结构化字段记录、日志查询、审计查询、请求状态查看和运行期调试。普通日志用于定位运行问题，审计日志用于追踪命令拒绝、工具调用、Cron 投递、持久化错误等关键行为。对于长期运行的机器人，日志与审计能力可以显著降低排障成本，也便于回溯高风险操作。

### 5. 常驻记忆与持久记忆分层

ElBot 将记忆分为轻量常驻记忆和可查询的持久记忆。常驻记忆只保存短小、稳定、真正需要每轮注入的信息，减少 token 消耗；更长、更复杂的记忆不强制自动注入，而是交给 LLM 在需要时主动查询。相比自动 RAG 或图检索长期记忆，这种方式更可控，避免不相关记忆污染上下文，也降低了检索和注入成本。

### 6. 普通 Cron 与 LLM Cron

ElBot 内置 Cron Runtime 和 LLM 可编排 Cron 服务。普通 Cron 可以按计划直接发送固定内容；LLM Cron 则允许用任务描述驱动模型执行周期性或一次性任务，并要求模型返回可解析的 JSON 结果。系统支持 once/周期任务、missed once 补投递、多平台通知、执行日志与失败提醒，适合提醒、日报、监控摘要和后台自动化任务。

### 7. ELyph：面向 LLM 协作的任务表示

ElBot 引入 ELyph Task Notation，用于描述 LLM Cron 与原生 skill。ELyph 的目标是减少自然语言任务描述中的歧义，用更短、更稳定的结构表达输入、输出、步骤、条件和约束。相比随意 Markdown，ELyph 更适合 LLM 之间复用和传递任务，也便于 lint、审计和后续工具化处理。

### 8. 可由 LLM 创建的 El Skill

ElBot 内置 `create_el_skill` 元工具，允许 LLM 将可复用经验沉淀为 El Skill。一个 El Skill 可以使用 ELyph 描述用途、输入输出和执行规则，并可选配套 Go 源码实现具体逻辑。创建前会经过 ELyph 校验，创建成功后自动 reload。这样，Agent 不只是一次性完成任务，还能把反复出现的工作固化为可复用能力。

### 9. 兼容互联网 Python Skill

除了原生 El Skill，ElBot 也兼容常见的 Python 外置 skill 结构。系统可以扫描 Python skill 的 `SKILL.md` 或 `SKILL.elyph`，读取名称、描述、适用场景和风险等级，并通过隐藏包装工具执行。这样既能复用互联网上已有的技能生态，又能逐步迁移到更结构化、更适合 ElBot 的 ELyph skill。

### 10. 多平台与富输出抽象

ElBot 抽象了平台层与输出层，支持 CLI 和 QQ OneBot，并预留扩展其他平台的空间。Hook、Tool 和插件不直接依赖具体平台发送消息，而是返回平台无关的输出意图，例如文本、图片、文件、at 或表情。平台支持富输出时按原格式发送，不支持时自动 fallback 为文本，降低插件与平台之间的耦合。

### 11. 会话、Fork 与上下文压缩

ElBot 内置持久化 Session 服务，支持会话恢复、归档、置顶、删除、分页查看和平台隔离。对于引用历史 assistant 消息等场景，可以自动 Fork 出分支会话，避免污染原时间线。上下文管理层支持按模型窗口解析、压缩 checkpoint 和摘要注入，在长对话中尽量控制上下文体积，并保留可追溯的历史结构。

### 12. 安全策略与风险确认

ElBot 的工具系统内置风险等级、权限判断和高风险确认流程。普通用户只能发现和调用低风险范围内的工具，超级管理员调用高风险工具时也需要确认。Shell 等高风险能力会经过风险分类和确认阶段，后台 Cron 还会使用轻量 sandbox 限制路径逃逸。这让 Agent 能拥有较强执行能力，同时尽量降低误操作风险。
